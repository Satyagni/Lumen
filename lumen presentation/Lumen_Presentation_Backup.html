Under The Hood · AI Engine
Cellpose: How the CNN Segments Cells
Not just a threshold, but a generalizable deep learning framework trained on 70,000+ segmented biological images.
🧠

U-Net Style Encoder-Decoder

Cellpose uses a U-Net architecture with residual blocks. The encoder downsamples the image through convolutional layers with ReLU activations, learning hierarchical spatial features. Skip connections feed encoder feature maps directly to the decoder — preserving fine-grained boundary detail at every scale.

🌊

Gradient Flow Field Prediction

Instead of predicting binary masks, Cellpose predicts a spatial gradient flow field — the direction each pixel should "flow" toward the center of its parent cell. A separate branch predicts cell probability. This topology-aware representation handles touching, overlapping, and irregular cells that binary segmentation fails on.

⚙️

Tunable at Inference Time

Lumen exposes flow_threshold and cellprob_threshold as user-selectable quality presets. Balanced preset targets maximum F1 on typical fluorescence. Sensitive preset lowers both thresholds for sparse or dim cell populations.

Why not classical watershed? Watershed from markers requires reliable seed detection: it fails on uneven illumination and touching cells. Cellpose's flow-field approach has no seed detection step; it handles confluent monolayers, irregular morphologies, and mixed populations without manual tuning.

Inside Lumen · Batch Engine
Folder-Scale Batch Processing
Point Lumen at a folder and walk away; the pipeline scales from 1 to 200+ images with zero supervision.
Lumen · Batch
Non-blocking

Persistent Background Worker

Cellpose inference runs on a dedicated QThread. The GUI progress view, live log, and cancel button all stay fully interactive during an active 200-image run.

Resume-safe

SKIPPED_ALREADY_EXISTS

Output file detection skips already-processed images on re-run. An interrupted 200-image batch picks up exactly where it left off with no wasted compute.

Memory-managed

Per-iteration GC

Each image's mask array is explicitly freed from memory after output is written. Prevents RAM accumulation across large runs on workstations with limited VRAM/RAM.

State machine

Pause / Resume / Cancel

Strict lifecycle FSM: IDLE → RUNNING → PAUSED → IDLE. Each transition is atomic, preventing torn states on interrupt.

Under The Hood · Architecture
System Design Principles
How Lumen enforces reliability, thread-safety, and extensibility.
🔬

Scientific Reliability First

Validated deep-learning frameworks over unproven classical heuristics. Cellpose <4.0.0 is pinned to prevent silent behavior changes in the segmentation model across updates.

📐

Simple > Clever

Flat module hierarchy, explicit Qt signals for every state transition, and structured JSON logging over clever abstractions. Every signal path is traceable in a debugger without indirection.

🔌

Extensible by Design

StarDist routing hooks already exist in the codebase. Adding a new segmentation backend means implementing one interface, requiring no changes to the GUI layer. The architecture is designed for the roadmap from day one.

🧵

Thread-Safety Pattern

Every worker signal uses Qt.QueuedConnection — enforced explicitly, not left to Qt's default resolution. This prevents the silent DirectConnection segfault that was patched during development.

Advanced Integrations · Planned
StarDist, OME-TIFF & Lab Infrastructure
Phase 2–3 integrations that connect Lumen to the broader scientific data ecosystem.
Segmentation

StarDist Backend

StarDist models star-convex polygons — optimal for densely packed spherical nuclei where Cellpose's flow field over-segments. Lumen will offer a side-by-side comparison UI: run both models on the same image and overlay results for researcher validation. StarDist hooks are already present in the codebase; wiring the thread worker is the only remaining step.

Export

OME-TIFF Native Export

OME-TIFF is the open standard for bioimaging metadata — embedding pixel calibration, channel names, Z/T coordinates, and instrument metadata in a single file. Direct OME-TIFF output from Lumen means results are immediately ingested by any downstream analysis tool (OMERO, Fiji Bio-Formats, QuPath) without format conversion.

Database

OMERO Integration

OMERO is the lab-scale image database standard. A planned Lumen-to-OMERO sync would push annotated masks, per-cell CSV data, and run manifests directly to a lab's OMERO server — enabling multi-user, multi-experiment data management without duplicating files on local storage.

AI

Local LLM Analysis Reports

Phase 5 targets a locally-running LLM (via llama.cpp or ollama) that reads the JSON run manifest and generates a structured methods section draft, including the exact model, parameters, cell counts, statistical summaries — in publication language. Patient data never leaves the workstation.

Design Constraint: Always Local

Every planned integration — StarDist, OME-TIFF, OMERO sync, and LLM reports, all running entirely on the researcher's workstation. No microscopy data ever transits a network we don't control. This is non-negotiable for clinical and patient-derived cell imaging.
